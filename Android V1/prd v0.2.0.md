## 2. 产品概述

**InlineNote** 是一款 Android 系统级辅助工具：在任意 App 中选中文字后，出现「解释」浮动按钮，点击后由 LLM 生成大白话解释，并以浮窗形式在原界面展示，无需复制、跳转或切应用。

**核心价值**：在阅读被专业术语/黑话阻断时，提供**原位、低摩擦、用户主动触发**的即时解释，不打断阅读流。

## 3. 当前进度与实现范围（v0.2.0）

### 3.1 已实现功能

| 模块 | 能力 | 说明 |
|------|------|------|
| **触发方式** | 文本选中 | 监听 `TYPE_VIEW_TEXT_SELECTION_CHANGED`，从节点或 `event.text` 取选中内容（兼容如 Chrome 等不暴露节点选中的场景） |
| | 复制即触发 | 监听剪贴板变化，复制后 800ms 防抖后显示「解释」按钮，视为“选中”的补充入口 |
| **过滤** | 长度与空白 | 选中内容 2～2000 字、非纯空白才显示触发按钮 |
| **触发 UI** | 浮动「解释」按钮 | 固定初始位置，可拖拽，点击后进入解释流程 |
| **解释生成** | 单轮 LLM | 仅传选中文本，不传前后文；接口 `LLMClient.explain(selectedText)`，实现为 `OpenAIClientDirect` |
| | 结果结构 | `ExplainResult`：大白话 + 要点拆解 + 可选潜台词，合并为一段展示文案 |
| **展示** | 解释浮窗 | 系统级浮窗，展示解释/错误文案，可关闭，不持久保存；宽度约 90% 屏宽 |
| **配置** | API Key / Base URL / Model | 主界面输入，EncryptedSharedPreferences（或回退普通 SP）本地加密存储；支持自定义 Base URL 与 Model（默认 gpt-4o-mini） |
| **权限与引导** | 无障碍 | 未开启时主界面说明 + 跳转系统无障碍设置 |
| | 悬浮窗 | Android 6+ 未授权时提示并跳转悬浮窗权限设置 |
| **错误处理** | 失败展示 | 浮窗内展示人话错误（如「网络不可用，未能生成解释」），提供「关闭」；无 API Key 或无悬浮窗时引导回主界面 |

### 3.2 技术栈与约束（与 V1 决策一致）

- **minSdk 24**，**targetSdk 34**；包名 `com.inlinenote.android`。
- **V1 仅对显式选中文本做解释**，不做前后文、不自动判断“是否看不懂”。
- **API Key 仅存于本机**，不写死、不提交仓库。
- **失败时不沉默**：浮窗内提示 + 关闭，不做重试按钮。

## 4. 用户故事与主流程

### 4.1 主流程（Happy Path）

1. 用户在主界面配置并保存 API Key（可选填 Base URL、Model），按引导开启无障碍与悬浮窗权限。
2. 在任意可选中文字的应用中**选中一段文字**（或**复制**该段文字）。
3. 出现浮动「解释」按钮。
4. 用户点击「解释」。
5. 浮窗展示加载/等待后展示大白话解释（含要点与潜台词若存在）；用户点击「关闭」回到阅读。

### 4.2 分支与异常

- **未配置 API Key**：点击「解释」后浮窗提示错误，并跳转主界面引导配置。
- **未开启悬浮窗**：同上，跳转主界面并引导开启悬浮窗权限。
- **网络/LLM 失败**：浮窗展示人话错误文案 +「关闭」，无重试。
- **选中过短/过长/空白**：不显示「解释」按钮。

## 5. 功能需求清单（v0.2.0 验收）

- [x] 无障碍服务监听文本选中事件，并可从 `event.text` 兜底获取选中内容。
- [x] 剪贴板监听 + 防抖，复制后显示「解释」按钮。
- [x] 选中内容长度与空白过滤（2～2000 字）。
- [x] 浮动「解释」按钮展示与点击响应，支持拖拽。
- [x] 主界面：API Key / Base URL / Model 输入与保存，本地加密存储。
- [x] 主界面：无障碍与悬浮窗权限说明及跳转设置。
- [x] 调用 LLM 单轮解释（仅选中文本），结果解析为 `ExplainResult` 并展示。
- [x] 解释浮窗展示正文 + 关闭；错误浮窗展示人话提示 + 关闭。
- [x] 无 Key / 无悬浮窗时引导回主界面。

---

## 6. 明确不做（v0.2.0 非目标）

与 TechDoc / V1 决策一致，以下**不在本版本**：

- 自动判断用户是否“看不懂”。
- 前后文（前后句）输入。
- 批量/长文档分析、摘要。
- 用户画像与个性化。
- Agent 自动执行或工具调用。
- 将解释写回原 App 或替换原文。
- 浮窗内重试按钮或自动重试。

---

## 7. 成功判据（可体验）

- 用户可在**数秒内**完成「选中 → 点解释 → 看到解释 → 关闭」并回到阅读。
- 首次使用通过主界面引导即可完成配置与权限，无需额外教程。
- 失败时有明确、人话的提示，且可关闭浮窗继续使用。

---

## 8. 后续迭代方向（非承诺）

在保持「原位、低摩擦、用户触发」的前提下，后续可能考虑：

- 多解释模式（如仅大白话 / 潜台词 / 怎么回应）。
- 轻量前后文扩展（前后一句）。
- 增加后端代理实现（`LLMClient` 另一实现），便于国内网络与鉴权。
- UI 与可访问性优化（仍保持“先可用”原则）。
