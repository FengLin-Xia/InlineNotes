## 1. 产品概述

### 1.1 产品定义

**InlineNotes** 是一款面向阅读场景的 AI 点读解释 SDK，能够嵌入到各类阅读应用（电子书阅读器、漫画App、新闻客户端等），为用户提供「一点即解释」的即时知识获取体验。

### 1.2 核心能力

用户在学习/阅读过程中，**选中任意文字**即可获得：
- 词语解释
- 背景知识
- 跨语言翻译
- 概念扩展

### 1.3 目标用户

| 优先级 | 用户群体 | 场景 |
|--------|----------|------|
| P0 | 外语阅读用户 | 看英文/日文书时点词翻译 |
| P0 | 学生/研究人员 | 阅读专业文献时理解术语 |
| P1 | 儿童/教育 | 辅助理解绘本、童书 |
| P2 | 泛阅读用户 | 了解冷门知识/典故 |

### 1.4 商业模式

**B2B SDK 授权**：
- 按调用量计费（CPQ）
- 包月套餐
- 定制化部署

## 2. 市场分析

### 2.1 市场规模

| 维度 | 数据 |
|------|------|
| 中国电子书用户 | 5亿+ |
| 有声书/阅读用户 | 3亿+ |
| AI阅读工具赛道 | 快速增长中 |

### 2.2 竞品格局

| 竞品 | 优势 | 劣势 |
|------|------|------|
| 微信读书AI | 生态好、用户多 | 仅限自家平台 |
| ChatPDF | 文档理解强 | 不支持选中即解释 |
| 有道/Google翻译 | 翻译准确 | 无知识扩展能力 |
| 系统自带词典 | 无需联网 | 功能单一 |

### 2.3 机会窗口

1. **阅读平台碎片化** — 没有统一的标准点读 SDK
2. **AI能力成熟** — 大模型 API 成本下降
3. **教育/出海需求** — 外语阅读是真实痛点

## 3. 产品定位

### 3.1 差异化定位

| 维度 | 竞品 | InlineNotes |
|------|------|-------------|
| 响应速度 | 慢（云端） | 快（缓存+本地小模型） |
| 知识深度 | 单一翻译 | 解释+背景+扩展 |
| 集成门槛 | 高 | 低（SDK 3行代码嵌入） |
| 定制能力 | 无 | 支持品牌/知识库定制 |

### 3.2 核心价值主张

> **「让阅读中的每一个疑问，都能在 1 秒内得到解答」**

## 4. 功能需求

### 4.1 功能清单

#### P0 - 必须实现

| 功能 | 描述 | 技术方案 |
|------|------|----------|
| 选中触发 | 用户选中文本后弹出解释按钮 | Android ViewOverlay |
| 词语解释 | 返回词义、例句、同义词 | 词典 API + 大模型 |
| 跨语言翻译 | 支持 10+ 语言互译 | 翻译 API |
| 缓存机制 | 本地缓存已查询词汇 | Room DB |
| SDK 集成 | 3 行代码接入 | Maven/Gradle |

#### P1 - 应该实现

| 功能 | 描述 | 技术方案 |
|------|------|----------|
| 知识卡片 | 展示更丰富的解释（图片、音频） | 大模型生成 |
| 历史记录 | 查看历史查询 | 本地存储 |
| 生词本 | 导出到背单词 App | API 导出 |
| 多平台 | iOS SDK | Swift |

#### P2 - 可以实现

| 功能 | 描述 |
|------|------|
| AI 问答 | 选中整段可追问 |
| 知识图谱 | 展示关联概念 |
| 离线模式 | 下载语言包离线查词 |

### 4.2 用户流程

```
用户阅读 → 选中文字 → 弹出「解释」按钮 → 点击 → 1秒内展示解释 → 用户关闭/继续阅读
```

### 4.3 界面交互

| 状态 | 设计 |
|------|------|
| 初始 | 选中文本后，文字上方显示「✨解释」按钮 |
| 加载中 | 按钮变为加载动画（<1秒） |
| 结果展示 | 底部弹出卡片：词义 + 例句 + 扩展 |
| 无结果 | 「未找到解释」+ 建议用户手动搜索 |

## 5. 技术方案

### 5.1 架构设计

```
┌─────────────────────────────────────────┐
│              阅读 App (Host)            │
├─────────────────────────────────────────┤
│           InlineNotes SDK               │
│  ┌─────────┐  ┌─────────┐  ┌────────┐ │
│  │ UI Layer│  │Service  │  │Cache   │ │
│  │(Overlay)│  │(LLM/API)│  │(Room)  │ │
│  └─────────┘  └─────────┘  └────────┘ │
├─────────────────────────────────────────┤
│           外部服务                      │
│  ┌─────────┐  ┌─────────┐  ┌────────┐ │
│  │ 词典API │  │翻译API  │  │LLM API │ │
│  └─────────┘  └─────────┘  └────────┘ │
└─────────────────────────────────────────┘
```

### 5.2 核心模块

| 模块 | 职责 | 技术 |
|------|------|------|
| InterceptService | 监听文本选中事件 | ContentObserver |
| ExplanationEngine | 决定用哪个服务解释 | 本地规则 |
| CacheManager | 缓存管理 | Room + LRU |
| SDK Bridge | 与宿主 App 通信 | AIDL |

### 5.3 性能指标

| 指标 | 目标 |
|------|------|
| 首次加载 | < 500ms |
| 解释响应 | < 1s (网络良好) |
| 内存占用 | < 30MB |
| 包体积 | < 2MB |

## 7. 竞争壁垒

### 7.1 技术壁垒

- **响应速度** — 缓存 + 本地小模型
- **准确性** — 多源融合（词典 + 大模型）
- **兼容性** — 覆盖主流 Android 版本

### 7.2 数据壁垒

- **用户查询数据** — 持续优化解释质量
- **行业知识库** — 垂直领域（文学/科学/历史）

### 7.3 商务壁垒

- **SDK 迁移成本** — 嵌入后替换成本高
- **合作案例** — 背书大客户

## 8. 风险与对策

| 风险 | 等级 | 对策 |
|------|------|------|
| 大厂免费入场 | 中 | 做深垂直场景 |
